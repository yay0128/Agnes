# Agnes AI Nodes for ComfyUI

Two custom nodes that wire Agnes's text and video models into ComfyUI:

| Node | Purpose |
| --- | --- |
| **Agnes Text (2.0 Flash)** | Expands a short user prompt into a cinematic video prompt |
| **Agnes Video Generate (V2.0)** | Submits the expanded prompt to Agnes Video V2.0, polls until done, downloads the MP4 |

The two together form the bundled `agnes_text_to_video.json` workflow.

## Install

```bash
# 1. Locate your ComfyUI custom_nodes directory
COMFYUI_NODES=~/path/to/ComfyUI/custom_nodes

# 2. Copy this folder
cp -R agnes_api "$COMFYUI_NODES/"

# 3. Make sure the `requests` package is available to ComfyUI's Python
#    (it ships with most desktop installs; if not: pip install requests)

# 4. Restart ComfyUI
```

The nodes will appear under **Agnes/text** and **Agnes/video** in the right-click node menu.

## Configure the API key

You can either:

- **Leave the `api_key` widget empty** and set the `AGNES_API_KEY` environment variable before launching ComfyUI (recommended), or
- **Paste your key** into the `api_key` widget on each node (it's a password field)

The widget value, when non-empty, takes precedence over the env var.

## Workflow

`../workflow.json` (in this directory's parent) is a ready-to-run graph that
connects the two nodes. To use it:

1. Open ComfyUI in a browser.
2. Drag `workflow.json` from your file manager onto the ComfyUI canvas.
3. (Optional) Edit the `user_prompt` widget on **Agnes Text (2.0 Flash)**.
4. Click **Queue Prompt**.

The video node will block for several minutes (typical: 2–6 min) while the
task runs on Agnes's side, then download the MP4 to the `output_dir` you
configured on the node (default `outputs/agnes` relative to ComfyUI's
working directory).

## Inputs

### Agnes Text (2.0 Flash)

| Input | Type | Default | Notes |
| --- | --- | --- | --- |
| `user_prompt` | STRING (multiline) | `"A cat walking on a beach at sunset"` | Your short idea |
| `api_key` | STRING (password) | `""` | Optional — falls back to `AGNES_API_KEY` env |
| `system_prompt` | STRING (multiline) | cinematic prompt-engineer prompt | Override to change the expansion style |
| `max_tokens` | INT | 512 | 64–2048 |
| `temperature` | FLOAT | 0.7 | 0.0–2.0 |

**Output:** `expanded_prompt` (STRING)

### Agnes Video Generate (V2.0)

| Input | Type | Default | Notes |
| --- | --- | --- | --- |
| `prompt` | STRING (forceInput) | — | Connect to `expanded_prompt` from Agnes Text |
| `api_key` | STRING (password) | `""` | Optional — falls back to `AGNES_API_KEY` env |
| `image` | STRING | `""` | Public URL for image-to-video |
| `height` | INT | 768 | 256–2048 |
| `width` | INT | 1152 | 256–2048 |
| `num_frames` | INT | 121 | Must be 81, 121, 161, 241, or 441 (8n+1, ≤ 441) |
| `frame_rate` | INT | 24 | 1–60 |
| `poll_interval` | INT | 10 | seconds between status polls |
| `max_wait` | INT | 1800 | seconds before giving up |
| `output_dir` | STRING | `outputs/agnes` | Where the MP4 is written |

**Output:** `video_path` (STRING, absolute path to the downloaded MP4)

## Notes

- Both endpoints are subject to 429 (upstream saturated) responses during
  peak load. The text node retries up to 5 times with backoff; the video
  node's create step also retries, and the poll loop tolerates 429 too.
- `num_frames` must satisfy the `8n + 1` rule and be ≤ 441. The node
  validates this client-side.
- `output_dir` is created if it doesn't exist.
- The video download uses streaming with 64 KB chunks — no large in-memory
  buffers.
